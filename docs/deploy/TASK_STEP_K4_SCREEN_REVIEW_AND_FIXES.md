## 공통 수정

### A. 분석하기 클릭 시 에러 화면
첫 번째 이미지에서 보이는 에러는 UI가 아니라 style 충돌이다.

에러:
- `Removing a style property during render (borderColor) when a conflicting property is set (border)`

위치:
- `features/upload/components/photo-upload-screen.tsx`
- `PhotoUploadScreen`
- flow step nav button 근처

수정:
- 동일 element에서 `border` shorthand와 `borderColor` non-shorthand를 섞지 말 것
- `border`, `borderColor`, `borderWidth`, `borderStyle`를 혼용하지 않게 정리
- step button style merge 시 활성/비활성 스타일을 안전하게 합치기

목표:
- 분석하기 클릭 시 스타일 경고/에러 없이 step 전환 가능해야 함

## 화면별 수정 요구

## 1. 사진 올리기 화면

현재 문제:
- 상단 `무엇부터 할까요?` 문구가 계속 남아 있음
- 이 문구는 첫 시작 화면에서만 필요하고, 이후 단계 화면에서는 삭제되어야 함
- `이전 결과 보기`, `촬영 가이드`가 아직 메인 업로드 카드 안에 남아 있는 느낌이 강함

수정 요구:
1. `무엇부터 할까요?`는 첫 시작 화면에서만 보이게
2. 사진 올리기 step에 진입한 이후에는 해당 문구 제거
3. `처음 선택으로 돌아가기` 옆 또는 같은 보조 액션 라인에 아래 항목 배치
   - 이전 결과 보기
   - 촬영 가이드
4. 업로드 카드 안의 보조 액션 느낌을 줄이고, 상단 보조 액션으로 정리

권장 상단 구조:
```text
처음 선택으로 돌아가기 | 이전 결과 보기 | 촬영 가이드

[사진 올리기] [분석 결과] [변경하기] [완성]
```

## 2. 분석 결과 화면

현재 문제:
- 바로 쓰는 영역 상단 문구가 이전 문맥을 일부 끌고 감
- `직접 꾸미기에 적용`은 목적이 모호함
- 더 보기 영역에 `직접 꾸미기` 버튼이 남아 있으면 단계 구조를 흐림
- `이 이미지로 판매하기`는 완성 단계 성격에 더 가까움

수정 요구:
1. 분석 결과 화면 상단 문구는 아래 흐름으로 정리
   - 제목: `분석 결과`
   - 설명: `사진을 기준으로 생성된 분석 결과를 한곳에서 확인할 수 있습니다.`
2. `직접 꾸미기에 적용` 버튼명은 `문구 변경`으로 수정
3. 더 보기 영역의 `직접 꾸미기` 버튼 삭제
4. `이 이미지로 판매하기` 영역은 분석 결과 화면에서 제거하고 완성 탭으로 이동

분석 결과 단계의 역할:
- 결과 확인
- 바로 저장/복사
- 다음 단계로 이동

## 3. 변경하기 화면

현재 문제:
- 미리보기가 길게만 보여 정확히 보기 어렵다
- 사진 느낌 변경 / 문구 변경 구조가 제대로 분리되지 않음
- 사진 쪽의 스타일이 사라진 느낌
- 문구 변경의 개념도 혼란스러움
- 지금처럼 한 화면에 다 펼치기보다, 미리보기 하단 탭으로 나누는 방식이 더 적절함

### 변경하기 단계의 핵심 구조
변경하기 화면에서는 **미리보기 아래 탭 버튼으로 영역을 전환**해야 한다.

권장 탭:
- 사진 변경
- 문구 변경

즉 현재 `사진느낌 / 문구느낌 / 문구변경` 식의 3분할이 아니라 아래처럼 2분할로 간다.

```text
[사진 변경] [문구 변경]
```

---

### 3-A. 사진 변경 탭

여섯 번째 이미지 방향처럼 구성

포함할 것:
- 분위기
- 스타일
- 프리셋(필요 시 스타일/프리셋 연계)
- 미리보기

정리:
- 사진 변경 = 이미지 인상 조정
- 스타일(예: 흑백, 필름, 빈티지)은 반드시 여기 포함
- 사진 스타일이 사라지지 않게 분명히 노출

권장 항목:
- 사진 분위기
- 사진 스타일

---

### 3-B. 문구 변경 탭

일곱 번째 이미지 방향처럼 구성

포함할 것:
- 말투 선택
- 감정 선택
- 문구 리스트
- 캡션 리스트
- 해시태그

정리:
- 문구 변경 = 텍스트 성격 조정
- 감정 / 말투는 여기 포함
- 스타일은 문구가 아니라 사진 쪽에 둔다

---

### 미리보기 수정

현재 문제:
- 미리보기가 가로로 길게만 보여 텍스트 포함 결과를 판단하기 어렵다

수정 요구:
1. 미리보기 비율을 실제 저장용 결과 판단이 가능한 형태로 조정
2. 너무 긴 배너형 미리보기 대신, 카드형 또는 저장 결과에 가까운 비율 우선
3. 탭 전환 시 미리보기는 유지하고, 아래 편집 영역만 바뀌게

## Codex 실행 프롬프트

현재 STEP K 화면을 실제 사용자 리뷰 기준으로 다시 정리해.

핵심 문제:
1. 분석하기 클릭 시 step nav button 쪽 스타일 충돌 에러가 남아 있다
2. 사진 올리기 화면 상단 문구와 보조 액션 위치가 어색하다
3. 분석 결과 화면에 `직접 꾸미기에 적용`, `더 보기 영역의 직접 꾸미기`, `이 이미지로 판매하기`가 잘못 남아 있다
4. 변경하기 화면은 미리보기 아래 탭으로 `사진 변경 / 문구 변경`을 나눠야 한다
5. 완성 탭에는 판매 연결 영역까지 포함되어야 한다
