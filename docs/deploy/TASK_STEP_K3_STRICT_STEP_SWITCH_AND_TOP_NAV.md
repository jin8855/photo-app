# TASK_STEP_K3_STRICT_STEP_SWITCH_AND_TOP_NAV.md

## 목적
현재 STEP K 구현이 “단계 전환형”이 아니라 “섹션 추가형”으로 들어가 화면이 길어지고 있다.
이번 작업은 이를 바로잡아, **현재 step의 핵심 화면만 보이는 구조**로 재정리하는 것이다.

---

## 현재 문제

현재 구현은 HOLD다.

문제:
1. `사진 분석하기`를 눌러도 업로드 영역이 그대로 남아 있다
2. 단계 버튼이 상단 네비게이션 역할을 하지 못한다
3. 결과 / 변경 / 완성이 아래로 계속 붙어 화면이 길어진다
4. 사용자가 지금 어느 단계에 있는지 한눈에 이해하기 어렵다

즉, 지금 구조는 “단계형 UX”가 아니라 “긴 단일 페이지 + 앵커 느낌”에 가깝다.

---

## 이번 작업의 핵심

이번 수정 목표:
- STEP K를 “섹션 추가형”이 아니라 “단계 전환형”으로 다시 적용
- 현재 step에 해당하는 핵심 화면만 보이게 정리
- 분석 결과 step에 들어가면 업로드 카드/업로드 버튼은 숨김
- 변경하기 step에 들어가면 분석 결과 중심 카드도 숨김
- 완성 step에 들어가면 편집 중심 카드는 숨기고 완료 CTA 중심으로 노출

---

## 상단 구조 요구

상단 구조는 아래 순서로 고정한다.

```text
처음 선택으로 돌아가기

[사진 선택] [분석 결과] [변경하기] [완성]
```

규칙:
- `처음 선택으로 돌아가기`는 최상단
- 그 아래에 단계 네비게이션 고정
- 현재 단계는 활성 표시
- 진입 불가 단계는 비활성 또는 조건부 활성
- 단계 버튼은 “스크롤 이동용”이 아니라 “현재 화면 전환용”이어야 함

---

## step 노출 원칙

### select
보여줄 것:
- 이미지 선택
- 사진 올리기
- 보조 액션: 이전 결과 보기, 촬영 가이드

숨길 것:
- 분석 결과 카드
- 변경하기 카드
- 완성 카드

---

### analyze
보여줄 것:
- 바로 쓰는 영역
- 더 보기 영역
- 기본 저장/복사 CTA
- 다음: 변경하기

숨길 것:
- 업로드 카드
- 이미지 선택 / 사진 올리기 버튼
- 변경하기 상세
- 완성 상세

---

### edit
보여줄 것:
- 사진 바꾸기
- 문구 바꾸기
- 자세히 꾸미기(접힘)
- 다음: 완성하기

숨길 것:
- 업로드 카드
- 분석 결과 중심 카드
- 완성 CTA 상세

---

### finalize
보여줄 것:
- 결과 이미지 만들기
- 생성 완료 상태
- 완성 이미지 저장
- 캡션 세트 복사
- 해시태그 복사
- 필요 시 패키지 다운로드

숨길 것:
- 업로드 카드
- 분석 결과 카드
- 과한 편집 카드

---

## 사진/문구 분리 원칙

변경하기 단계에서는 아래처럼 분리한다.

### 사진 바꾸기
- 분위기
- 스타일
- 프리셋
- 미리보기

### 문구 바꾸기
- 말투 선택
- 감정 선택
- 문구 리스트
- 캡션 리스트
- 해시태그

중요:
- “느낌 변경”처럼 뭉뚱그린 표현을 줄일 것
- 사용자가 “이미지 조정”과 “텍스트 조정”을 바로 구분할 수 있어야 함

---

## 구현 원칙

1. 한 화면에 모든 단계를 길게 이어붙이지 말 것
2. 현재 step의 화면만 중심 노출할 것
3. 이전 단계 요소는 필요한 최소 정보 외에는 숨길 것
4. step state 조건부 렌더링으로 해결할 것
5. API / 분석 / 생성 로직은 건드리지 말 것
6. 라우팅 분리는 하지 말 것

예시 상태:
- `currentStep: 'select' | 'analyze' | 'edit' | 'finalize'`

문제:

- 단계 버튼이 상단 네비게이션 역할을 하지 못하고 있다
- 결과/변경/완성이 아래로 계속 붙어 화면이 길어진다

이번 수정 목표:
- STEP K를 “섹션 추가형”이 아니라 “단계 전환형”으로 다시 적용해
- 현재 step에 해당하는 핵심 화면만 보이게 해
- 분석 결과 step에 들어가면 업로드 카드/업로드 버튼은 숨겨
- 변경하기 step에 들어가면 분석 결과 중심 카드도 숨겨
- 완성 step에 들어가면 편집 중심 카드는 숨기고 완료 CTA 중심으로 보여줘

상단 구조 요구:
1. `처음 선택으로 돌아가기`를 최상단에 둔다
2. 그 아래에 단계 네비게이션을 둔다
   - 사진 선택
   - 분석 결과
   - 변경하기
   - 완성

step 노출 원칙:
- select: 업로드 화면만
- analyze: 분석 결과 화면만
- edit: 변경하기 화면만
- finalize: 완성 화면만

추가 구조 요구:
- 변경하기 단계는 아래처럼 분리
  - 사진 바꾸기: 분위기, 스타일, 프리셋
  - 문구 바꾸기: 말투 선택, 감정 선택, 문구/캡션 리스트

중요:
- 한 화면에 모든 단계를 길게 이어붙이지 말 것
- 현재 step의 화면만 중심 노출할 것
- 이전 단계 요소는 필요한 최소 정보 외에는 숨길 것

수정 범위:
- step state 기반 조건부 렌더링
- 상단 단계 네비게이션 위치 재배치
- 각 단계별 화면 노출 재정리
- 필요한 최소 문구 보정

절대 건드리지 말 것:
- API / 분석 로직 / 생성 로직
- Tone / Style / Theme 내부 구조
- 페이지 라우팅 분리

결과:
1. 변경 내용
2. PASS / HOLD
3. 다음 단계 1개
