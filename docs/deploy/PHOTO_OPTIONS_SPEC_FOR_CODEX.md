# PHOTO_OPTIONS_SPEC_FOR_CODEX.md

## 목적
변경하기 화면에서 사진/문구 옵션이 UI만 있고 실제 의미가 흐려지지 않도록, 옵션 체계를 먼저 고정한다.

이번 문서는 **정의서**다.
Codex는 이 문서를 기준으로 라벨, 선택 그룹, 상태 연결, 미리보기 반영을 맞춘다.

---

# 1. 최상위 구조

변경하기 단계는 아래 2개 영역으로 나눈다.

## A. 사진 변경
사진 자체의 시각 표현을 바꾸는 영역

포함:
- 분위기
- 사진 스타일
- 프리셋
- 자세히 꾸미기(고급 조정)

## B. 문구 변경
문장/캡션/해시태그의 표현을 바꾸는 영역

포함:
- 말투 선택
- 감정 선택
- 이미지 문구
- 대표 캡션
- 자세히 보기(리스트/해시태그 팝업)

---

# 2. 사진 변경 옵션 정의

## 2-1. 분위기
### 의미
사진의 전체 인상과 방향을 정하는 상위 선택

### 사용자 노출명
- 분위기

### 예시 옵션
- 차분하게
- 밝게
- 선명하게
- 부드럽게
- 따뜻하게
- 시원하게

### 반영 방식
- 선택 즉시 미리보기 반영
- 색감, 대비, 밝기, 채도, 톤 방향값에 영향

### 주의
- 분위기는 사진용이다
- 문구 쪽 감정/말투와 섞지 않는다

---

## 2-2. 사진 스타일
### 의미
사진의 시각 스타일을 직접 바꾸는 선택

### 사용자 노출명
- 사진 스타일

### 예시 옵션
- 원본
- 흑백
- 필름
- 빈티지
- 소프트
- 선명
- 웜톤
- 쿨톤

### 반영 방식
- 선택 즉시 미리보기 반영
- filter / overlay / color transform / preset tone에 직접 연결

### 주의
- “사진 스타일”은 분명히 보여야 한다
- “사진 필터”보다 사용자 기준으로 더 직관적이다

---

## 2-3. 필터
### 의미
스타일 내부의 기술적 조정값 또는 세부 표현

### 권장 방향
- 기본 화면에서는 “사진 스타일”로 통합
- 고급 사용자만 자세히 꾸미기에서 필터 세부 조정

### 예시 항목
- 밝기
- 대비
- 채도
- 온도
- 그림자
- 비네트
- 입자감

### 주의
- 스타일과 필터를 같은 레벨로 동시에 크게 보여주면 헷갈린다

---

## 2-4. 프리셋
### 의미
분위기 + 스타일 + 일부 텍스트 배치까지 포함하는 “전체 묶음 적용”

### 사용자 노출명
- 프리셋

### 예시 옵션
- 감성 카드형
- 밝은 소개형
- 어두운 몰입형
- 새벽 블루
- 빈티지 크림
- 선셋 글로우

### 반영 방식
- 선택 즉시 미리보기 반영
- 내부적으로 아래를 묶어서 적용 가능
  - 분위기
  - 사진 스타일
  - 필터 강도
  - 글씨 배치/톤 일부

### 주의
- 프리셋은 “한 번에 적용” 의미가 분명해야 한다
- 선택 시 적용됨 상태가 반드시 보여야 한다

---

## 2-5. 자세히 꾸미기
### 의미
고급 사용자용 세부 조정

### 사용자 노출명
- 자세히 꾸미기

### 포함 예시
- 글씨 위치
- 폰트
- 크기
- 굵기
- 색상
- 그림자
- 배경 박스
- 줄 간격
- 좌표 조정
- 고급 필터

### 노출 방식
- 기본 접힘
- 기본 사용자 흐름에서는 최소 노출

---

# 3. 문구 변경 옵션 정의

## 3-1. 말투 선택
### 의미
문장이 말하는 방식을 바꾸는 선택

### 사용자 노출명
- 말투 선택

### 예시 옵션
- 담백하게
- 부드럽게
- 강하게
- 감성적으로
- 또렷하게

### 반영 방식
- 선택 즉시 문구/캡션 preview 변경
- 이미지 자체는 바꾸지 않음

---

## 3-2. 감정 선택
### 의미
문장에 담기는 정서를 정하는 선택

### 사용자 노출명
- 감정 선택

### 예시 옵션
- 설렘
- 위로
- 따뜻함
- 유쾌함
- 그리움
- 벅참

### 반영 방식
- 선택 즉시 문구/캡션 preview 변경
- 이미지 자체는 바꾸지 않음

### 주의
- 감정은 문구용이다
- 사진 분위기와 섞지 않는다

---

## 3-3. 이미지 문구
### 의미
이미지 위에 크게 들어가는 대표 문구

### 사용자 노출명
- 이미지 문구

### 노출 방식
- 기본 화면에서 1개 선택값만 표시
- dropdown 또는 single selector 허용

### 추가 보기
- 자세히 보기 버튼에서 전체 리스트 노출

---

## 3-4. 대표 캡션
### 의미
본문/게시글에 함께 쓸 대표 캡션

### 사용자 노출명
- 대표 캡션

### 노출 방식
- 기본 화면에서 1개 선택값만 표시
- dropdown 또는 single selector 허용

### 추가 보기
- 자세히 보기 버튼에서 전체 리스트 노출

---

## 3-5. 해시태그
### 의미
게시글에 함께 쓸 해시태그 묶음

### 사용자 노출명
- 해시태그

### 노출 방식
- 기본 화면에서는 숨김 가능
- 자세히 보기 팝업에서 함께 노출 권장

---

## 3-6. 자세히 보기
### 의미
문구/캡션/해시태그 전체 목록을 확인하는 팝업 진입

### 사용자 노출명
- 자세히 보기

### 팝업 안에 포함할 것
- 문구 리스트
- 캡션 리스트
- 해시태그

### 주의
- 하단에 길게 리스트를 계속 보여주지 말 것
- 기본 화면은 짧고, 자세히 보기로 확장

---

# 4. 화면별 기본 노출 정책

## 변경하기 > 사진 변경 탭
기본 노출:
- 분위기
- 사진 스타일
- 프리셋
- 미리보기
- 자세히 꾸미기(접힘)

숨길 것:
- 긴 문구 리스트
- 해시태그 전체 리스트

---

## 변경하기 > 문구 변경 탭
기본 노출:
- 말투 선택
- 감정 선택
- 이미지 문구
- 대표 캡션
- 자세히 보기 버튼

팝업 노출:
- 문구 리스트
- 캡션 리스트
- 해시태그

---

# 5. 상태 연결 규칙

## 사진 변경
- 분위기 선택 -> preview 즉시 변경
- 사진 스타일 선택 -> preview 즉시 변경
- 프리셋 선택 -> preview 즉시 변경
- 선택된 항목은 활성 상태 표시

## 문구 변경
- 말투 선택 -> 문구 즉시 변경
- 감정 선택 -> 문구 즉시 변경
- 이미지 문구 선택 -> preview text 즉시 변경
- 대표 캡션 선택 -> 대표 캡션 즉시 변경

---

# 6. 금지 규칙

- 사진 옵션과 문구 옵션을 한 섹션에서 섞지 말 것
- “느낌 변경”처럼 추상적인 단일 묶음으로 되돌리지 말 것
- 스타일 / 필터 / 프리셋을 같은 위계로 복잡하게 나열하지 말 것
- 문구 리스트를 기본 화면 하단에 길게 노출하지 말 것
- 선택했는데 미리보기가 바뀌지 않는 상태를 만들지 말 것

---

# 7. Codex 실행 프롬프트

현재 문제:
- 사진 스타일, 프리셋, 분위기 정의가 UI에 명확히 고정되지 않아 반영이 흐려져 있다
- 문구 변경도 말투/감정/대표 문구/캡션의 역할이 섞여 있다

이번 작업:
- 첨부된 옵션 정의서를 기준으로 변경하기 화면의 옵션 구조를 다시 맞춰
- 사진 변경과 문구 변경을 분리하고,
- 각 옵션이 무엇을 바꾸는지 UI와 상태 연결에서 명확히 반영해

핵심 적용:
1. 사진 변경 탭
   - 분위기
   - 사진 스타일
   - 프리셋
   - 자세히 꾸미기(접힘)

2. 문구 변경 탭
   - 말투 선택
   - 감정 선택
   - 이미지 문구
   - 대표 캡션
   - 자세히 보기(팝업)

3. 필터
   - 기본 화면에서는 사진 스타일에 통합하거나
   - 자세히 꾸미기 안으로 이동

4. 모든 선택은 즉시 미리보기에 반영

절대 건드리지 말 것:
- API
- 분석 로직
- 생성 로직
- step 구조

결과:
1. 어떤 옵션 체계로 반영했는지
2. PASS / HOLD
3. 다음 단계 1개
