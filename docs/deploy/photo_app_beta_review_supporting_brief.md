## 1. 현재 프로젝트 평가 요약

### 전체 판단
- 구조는 베타 직전 수준으로 나쁘지 않다.
- provider / repository / storage 분리가 되어 있어 확장 가능성이 있다.
- 다만 사용자 첫 경험, AI 결과 신뢰도, 결과 화면의 과밀도가 핵심 리스크다.

### 한 줄 결론
**지금은 “기능이 부족한 프로젝트”가 아니라 “방향은 좋은데 첫 인상이 헷갈리는 프로젝트”에 가깝다.**

---

## 2. 현재 강점

### 2-1. 구조적 강점
- DB provider 분리
- storage adapter 분리
- analysis provider 분리
- phrase engine strategy 패턴 적용

### 2-2. 이미 잘 고친 부분
- history 이미지 preview URL 처리
- server roundtrip 위주의 미리보기 문제 완화
- client-side preview 전환
- action/speed 스타일 톤 보정

### 2-3. 서비스 방향 강점
- 사진 업로드 → 분석 → 문구 → 이미지 생성 흐름이 하나의 유스케이스로 이어짐
- SNS 업로드 직전 사용자에게 직접적인 가치가 있음

---

## 3. 핵심 문제

## 3-1. UX 문제
- 처음 온 사용자가 어디를 먼저 눌러야 하는지 직관적이지 않다.
- 결과 정보량이 많다.
- “좋은 기능이 많음”이 “복잡함”으로 느껴질 위험이 있다.

## 3-2. AI 결과 신뢰도 문제
- fallback classifier가 heuristic 기반이라 오판 가능
- 특정 사진에서 분위기/스타일이 어긋나면 서비스 전체 신뢰가 떨어질 수 있음

## 3-3. 카피 품질 편차
- 어떤 결과는 좋지만 어떤 결과는 번역투 느낌 가능
- 감성 카피와 실전형 카피가 섞여 사용자 취향 대응이 약함

## 5. 새 기능 제안 — 사진 개선 팁

### 왜 중요한가
이 기능은 단순 부가 기능이 아니라 서비스 포지션을 바꾼다.

기존:
- 사진을 보고 문구 생성

확장 후:
- 사진을 보고
  - 분위기 분석
  - 캡션 추천
  - 해시태그 추천
  - 텍스트 이미지 생성
  - 촬영 개선 코칭

즉, **캡션 생성기 → 사진 코칭 도구**로 진화한다.

---

## 6. 촬영 팁 기능 상세 방향

### 사용자에게 보여줄 항목
- 구도 팁
- 빛/노출 팁
- 색감/분위기 팁
- 다시 찍는 팁
- 기기별 팁

### 예시 문구
- 피사체를 중앙에서 조금만 벗어나게 두면 더 안정적으로 보입니다.
- 하이라이트가 강해서 밝은 부분이 날아간 느낌입니다. 노출을 조금 낮춰보세요.
- 전체 톤이 조금 탁해 보여서 더 밝은 시간대나 따뜻한 색감이 어울립니다.
- Galaxy Ultra에서는 3배 또는 5배 줌으로 찍으면 배경 정리가 더 자연스러울 수 있습니다.

## 7. 갤럭시 S25 Ultra / S26 Ultra 팁 방향

이번 1차 구현에서는 아래 원칙이 적절하다.

### 원칙
- 실제 EXIF 기반 자동 기기 판별에 집착하지 않는다.
- 사용자가 기기 타입을 선택할 수 있게 해도 된다.
- 세부 스펙 나열보다 “찍을 때 바로 쓸 수 있는 팁”이 중요하다.

### 추천 팁 주제
- 고배율 줌 활용 팁
- 야간 촬영 안정화 팁
- HDR/역광 활용 팁
- 음식/소품/인물 거리 조절 팁
- 흔들림 방지 팁
