# TASK_STEP_J3_MULTI_UPLOAD_AND_PRIMARY_SELECTION.md

## 목적
STEP J 1차에서 홈 2분기 구조를 PASS로 정리했고,
STEP J2에서 촬영 가이드의 `상황 -> 대상 -> 기기` 흐름도 PASS로 정리했다.

이제 다음 핵심 HOLD인
`단일 업로드 중심 흐름`을 개선한다.

이번 작업의 목표는
**여러 장 업로드 + 대표 사진 선택**
흐름을 가장 작은 범위로 추가해서,
실제 사용자 행동과 앱 흐름을 더 가깝게 맞추는 것이다.

---

## 현재 상태

현재 PASS 상태:
- 홈 첫 화면 2분기 구조
- 독립 촬영 가이드
- 촬영 가이드 `상황 -> 대상 -> 기기`
- 결과 화면 3단 구조
- photo tips 3~5개 제한 / 우선순위 정렬
- deviceType optional
- build / typecheck / photo style / regression test 통과

현재 남은 핵심 UX HOLD:
- 업로드 흐름이 단일 파일 중심
- 실제 사용자는 보통 여러 장을 찍고 그중 하나를 고른다
- 체크리스트 기준의 `여러 장 업로드 후 대표 사진 선택`이 아직 필요하다

---

## 이번 단계 범위

### 반드시 할 것
1. 여러 장 업로드를 허용한다
2. 업로드 후 대표 사진을 1장 고를 수 있게 한다
3. 분석 / 결과 생성은 대표 사진 기준으로 이어지게 한다
4. 사용자가 현재 어떤 사진이 대표인지 명확히 알 수 있어야 한다

### 이번 단계에서 허용되는 최소 확장
- 기본 추천 표시 또는 기본 선택 1장 지정
- 썸네일 리스트 / 카드형 선택 UI
- 대표 사진 변경 UI

---

아래는 이번 STEP J3에서 제외한다.

- 결과 화면 3단 구조 변경
- photo tips 생성 규칙 재설계
- Tone / Style / Theme 내부 구조 변경
- 촬영 가이드 구조 재작업

- AI 추천 점수 시스템 대규모 설계

즉, 이번 단계는
**여러 장을 올리고, 그중 대표 1장을 골라 분석으로 넘기는 것**
까지만 한다.

---

## 구조 규칙 (절대 유지)

다음 규칙은 유지한다.

- Tone / Style / Theme 3축 유지
- Frame은 Tone 내부
- features / domains / content 분리 유지
- UI는 로직을 가지지 않음
- 분석 로직은 대표 사진 입력 기준으로만 이어지게 한다
- 기존 결과 화면 3단 구조는 유지한다

---

## UX 목표

사용자는 다음처럼 느껴야 한다.

1. 사진 여러 장을 한 번에 올릴 수 있다
2. 어떤 사진이 대표인지 바로 보인다
3. 필요하면 다른 사진으로 쉽게 바꿀 수 있다
4. 결과는 내가 고른 대표 사진 기준으로 나온다

핵심은
**선택 자체가 어렵지 않아야 한다**는 것이다.

---

## 구현 방향

### 1. 최소 변경 우선
기존 업로드/분석 흐름을 전부 갈아엎지 말고,
입력부에 다중 업로드와 대표 선택 단계를 추가하는 방식으로 간다.

### 2. 대표 사진 기준 단일 분석 유지
분석 엔진을 여러 장 동시 분석으로 확장하지 않는다.
이번 단계에서는:
- 여러 장 업로드 가능
- 그러나 실제 분석은 대표 사진 1장 기준
으로 유지한다.

### 3. 대표 선택 UI는 직관적으로
가능하면 다음 중 하나로 구현한다.
- 썸네일 목록 + 선택 배지
- 카드 선택 + 대표 표시
- 라디오/체크 스타일은 가능하지만 시각적으로 분명해야 함

### 4. 기본 대표값 필요
여러 장 업로드 직후 아무것도 선택되지 않은 상태는 피한다.
기본값은 아래 중 하나로 둔다.
- 첫 번째 사진 자동 대표 선택
- 또는 기존 추천 기준이 있으면 추천 1장 자동 지정

처음 단계에서는 `첫 번째 자동 선택`도 허용된다.
중요한 건 **빈 상태를 만들지 않는 것**이다.

- 다중 사진 각각에 대해 결과를 동시에 만드는 구조
- 대표 사진과 분석 대상이 따로 노는 구조

## PASS 조건

아래를 만족하면 PASS다.

1. 사용자가 사진 여러 장을 업로드할 수 있다
2. 업로드 직후 대표 사진이 1장 정해져 있다
3. 사용자가 대표 사진을 다른 사진으로 바꿀 수 있다
4. 분석 / 결과 생성이 대표 사진 기준으로 동작한다
5. 현재 대표 사진이 UI에서 분명하게 보인다
6. 홈 2분기 / 촬영 가이드 / 결과 화면 3단 구조 / photo tips / deviceType 흐름이 깨지지 않는다

검증:
- `npm run typecheck`
- `npm run build`

가능하면 추가 확인:
- `npm run test:photo-styles`
- `npx tsx tests/photo-tips-regression.test.ts`

---

## HOLD 조건

아래 중 하나라도 있으면 HOLD다.

- 여러 장 업로드는 되지만 대표 사진이 불명확하다
- 대표 사진을 바꿔도 실제 분석 대상이 바뀌지 않는다
- UI가 복잡해져서 초보 사용자가 더 헷갈린다
- 기존 업로드 흐름보다 느리거나 무거워졌다
- 결과 화면 / 촬영 가이드 / 홈 구조에 불필요한 영향이 갔다
