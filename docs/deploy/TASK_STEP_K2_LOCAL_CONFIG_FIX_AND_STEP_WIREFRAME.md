# TASK_STEP_K2_LOCAL_CONFIG_FIX_AND_STEP_WIREFRAME.md

## 목적
로컬 테스트가 외부 배포에만 의존하지 않도록,
먼저 로컬 업로드 에러 원인을 정리하고 최소 수정으로 개발 환경을 복구한다.

동시에 STEP K의 4단계 UX를 기준으로
사진/문구 분리 구조까지 반영한 UI 와이어프레임을 확정한다.

---

## 현재 에러 해석

로컬 콘솔 에러 핵심:

- route: `/api/photos/upload`
- message: `Unexpected upload route failure.`
- cause: `configMissing:MSSQL_CONNECTION_STRING|LOCAL_SQLSERVER_INSTANCE`

즉, 현재 문제는 UI 자체보다 먼저
**로컬 업로드 API가 필요한 DB/로컬 SQL 설정을 못 찾아서 500이 나는 상태**다.

이 상태에서는:
- UI 수정이 반영돼도
- 업로드 후 다음 단계 진입이 막혀
- 실제로는 화면 변화가 안 된 것처럼 느껴질 수 있다

---

## 결론

### PASS
- 외부 배포 테스트는 계속 병행 가능
- 배포 환경 검증 자체는 유효

### HOLD
- 로컬을 포기하고 외부 배포만으로 개발하면 안 된다
- 현재 에러는 명확한 환경 설정 누락이므로 로컬을 먼저 복구하는 게 맞다

### 최종 판단
- **배포 테스트만으로 진행하지 말 것**
- **로컬 업로드 경로를 먼저 복구**
- 그 다음 STEP K UI 구조 반영 확인

## 로컬 에러 수정 가이드

Codex는 먼저 아래를 확인해야 한다.

### 확인 항목
1. `.env.local` 또는 로컬 환경 변수에 아래 중 하나가 있는지 확인
   - `MSSQL_CONNECTION_STRING`
   - `LOCAL_SQLSERVER_INSTANCE`

2. 업로드 API가 실제로 어떤 설정을 필수로 읽는지 확인
   - `/api/photos/upload`
   - config loader
   - DB connection factory 또는 upload repository 경로

3. 로컬에서 DB가 없을 때 fallback이 있는지 확인
   - mock storage
   - file/local storage
   - in-memory fallback

---

## 로컬 수정 목표

### 목표 A
로컬 SQL 설정이 있으면 정상 업로드 되게 한다

### 목표 B
로컬 SQL 설정이 없더라도 개발 모드에서는
명확한 안내 문구 또는 fallback 경로가 있어야 한다

### 하면 좋은 것
- 개발 환경에서만 친절한 config error 메시지 노출
- "로컬 업로드 설정이 없습니다" 수준의 설명 추가
- 무조건 500 stack만 보이지 않게 정리

# UI 와이어프레임

## 전체 4단계

```text
[1. 사진 선택] -> [2. 분석 결과] -> [3. 변경하기] -> [4. 완성]
```

---

## STEP 1. 사진 선택

```text
┌──────────────────────────────────────┐
│ 상단 진행 표시: 1 사진 선택          │
├──────────────────────────────────────┤
│ 제목: 사진을 올려 시작하세요         │
│ 설명: 사진 한 장 또는 여러 장 선택   │
│                                      │
│ [이미지 선택]                        │
│ [사진 올리기]                        │
│                                      │
│ ─ 보조 링크 ─                        │
│ 처음 선택으로 돌아가기               │
│ 이전 결과 보기                       │
│ 촬영 가이드                          │
└──────────────────────────────────────┘
```

### 원칙
- 메인 CTA는 2개만
- 보조 액션은 아래로 내린다
- 업로드 완료 후 `사진 분석하기` 표시

## Codex 실행 프롬프트

현재 상태:
- 로컬 테스트에서 `/api/photos/upload`가 500
- 원인은 `configMissing:MSSQL_CONNECTION_STRING|LOCAL_SQLSERVER_INSTANCE`
- 외부 배포는 정상 동작하지만 로컬은 설정 누락으로 막힘
- 동시에 STEP K 4단계 구조와 사진/문구 분리 UX를 반영해야 함

이번 작업:
1. 로컬 업로드 에러 원인을 먼저 정리하고 최소 수정으로 로컬 테스트를 복구해
2. STEP K 구조를 반영할 때, 변경하기 단계에서 사진과 문구를 명확히 분리해

핵심 요구:
- 배포만으로 개발하지 말고 로컬 업로드 경로를 먼저 복구
- 변경하기 단계는 아래처럼 분리
  - 사진 바꾸기: 분위기, 스타일
  - 문구 바꾸기: 말투 선택, 감정 선택, 문구/캡션 리스트
- 현재 기능/로직은 최대한 유지
- 페이지 분리는 하지 말고 step state 기반으로 간다

수정 범위:
- 로컬 config 확인 및 최소 복구
- STEP K UI 구조 반영
- 필요한 최소 문구 보정

절대 건드리지 말 것:
- API 전체 재설계
- 생성/분석 로직 변경
- Tone / Style / Theme 내부 구조 변경
- 라우팅 분리
