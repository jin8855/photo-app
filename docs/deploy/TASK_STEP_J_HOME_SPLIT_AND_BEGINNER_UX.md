# TASK_STEP_J_HOME_SPLIT_AND_BEGINNER_UX.md

## 목적
STEP I에서 HOLD로 남은 핵심 UX 항목을 가장 작은 범위로 정리한다.

이번 작업의 1차 목표는 기능 추가가 아니라, **첫 사용자 진입 흐름을 단순하게 만드는 것**이다.

---

## 현재 판단

STEP I 점검 결과 HOLD의 핵심은 아래 3가지다.

1. 홈이 아직 초보자 기준 2분기 구조가 아니다.
2. 촬영 가이드가 `상황 -> 대상 -> 기기`가 아니라 `상황 -> 기기` 구조다.
3. 업로드 흐름이 단일 파일 중심이라 실제 사용자 행동과 어긋난다.

이번 STEP J에서는 **전부 한 번에 크게 갈아엎지 말고**,  
가장 효과가 큰 순서대로 **홈 진입 구조 정리**를 먼저 수행한다.

---

## 이번 단계 범위

### 반드시 할 것
1. 첫 화면을 2분기 구조로 변경
   - `사진 찍었어요`
   - `지금 찍으려고요`

2. 각 진입 버튼의 목적을 명확히 연결
   - `사진 찍었어요` → 업로드 흐름
   - `지금 찍으려고요` → 촬영 가이드

3. 첫 화면에서 초보 사용자의 판단 부담을 줄이기
   - 기존 기능 나열형 UI를 줄이고
   - 행동 선택형 UI로 바꾼다

### 이번 단계에서 같이 정리 가능
4. 홈 문구/설명 간소화
5. 기존 버튼 배치 재정리
6. 히스토리/보조 기능은 첫 화면의 주 행동보다 약하게 배치

---

## 이번 단계에서 하지 말 것

아래는 이번 STEP J 1차 범위에서 제외한다.

- 다중 업로드 전체 구현
- 대표 사진 선택 UI 구현
- 추천 이미지 자동 선택 로직
- Tone / Style / Theme 내부 구조 변경
- 분석 로직 변경
- photo tips 생성 규칙 변경
- 결과 화면 3단 구조 변경
- deviceType contract 변경
- 대규모 라우팅 재설계

즉, 이번 단계는 **홈 UX 재구성**이 중심이다.

## 구조 규칙 (절대 유지)

다음 규칙은 유지한다.

- Tone / Style / Theme 3축 유지
- Frame은 Tone 내부 유지
- features / domains / content 분리 유지
- UI는 로직을 가지지 않음
- content를 UI에서 직접 다루지 않음
- 기존 결과 화면 3단 구조 유지

참고 기준:

## 목표 UX

사용자가 앱에 처음 들어왔을 때
생각보다 먼저 행동을 고르게 만든다.

### 기대 흐름
- 첫 화면 진입
- 두 개 중 하나만 선택
- 바로 해당 흐름으로 이동

### 1안
- 사진 찍었어요
  - 이미 사진이 있다
  - 업로드해서 결과 받기

### 2안
- 지금 찍으려고요
  - 아직 사진이 없다
  - 촬영 가이드 먼저 보기

---

## 홈 화면 요구사항

### A. 메인 행동 2개만 강하게 보이게
첫 화면에서 가장 강하게 보이는 요소는 아래 2개여야 한다.

- 사진 찍었어요
- 지금 찍으려고요

### B. 보조 기능은 약하게
아래 항목이 존재하더라도 메인 행동보다 약해야 한다.

- 히스토리
- 추가 설명
- 기타 보조 버튼

### C. 문장 길이 최소화
버튼 설명은 짧고 행동형으로 유지한다.

예시 문구 방향:

- 사진 찍었어요  
  찍은 사진을 올리고 바로 결과를 받아보세요

- 지금 찍으려고요  
  상황에 맞는 촬영 팁부터 보고 시작하세요

문구는 이 의미만 유지하면 되고, 실제 카피는 UI에 맞게 더 짧게 조정 가능하다.

---

## 라우팅 / 연결 요구사항

### 사진 찍었어요
- 현재 업로드 흐름 진입점으로 연결
- 기존 업로드 기능은 유지
- 업로드 폼 자체를 없애는 것이 아니라
  첫 진입 위치를 홈 2분기 다음으로 이동시킨다

### 지금 찍으려고요
- 기존 `/shooting-guide` 또는 현재 촬영 가이드 진입점으로 연결
- 기존 독립 촬영 가이드 구조를 깨지 않는다

## UI 원칙

### 해야 할 것
- 카드형 또는 2분기 CTA형으로 단순하게 구성
- 첫 사용자 기준으로 직관성 우선
- 모바일/좁은 폭에서도 2개 선택이 선명하게 보이게 구성

### 하지 말 것
- 첫 화면에 기능을 여러 개 동등하게 나열
- 홈에서 설정/고급 기능을 먼저 보여주기
- 첫 화면에 설명 텍스트를 길게 쓰기
- 홈에서 로직 분기를 복잡하게 넣기

## 구현 가이드

### 1. 최소 변경 우선
기존 페이지 구조를 전부 갈아엎지 말고,
홈 진입부만 정리해서 2분기 UX를 만든다.

### 2. 기존 기능 재사용
가능하면 아래는 기존 것을 재사용한다.
- 업로드 화면/컴포넌트
- 촬영 가이드 라우트/화면
- 히스토리 기능
- 결과 화면

### 3. 시각적 우선순위 분명히
메인 CTA 2개가 가장 먼저 보여야 한다.
보조 기능은 크기, 위치, 강조 모두 한 단계 낮춘다.

### 4. UI에 로직 넣지 말 것
버튼 클릭 후 어디로 보내는지 정도만 연결하고,
분석/추천/생성 관련 판단 로직을 홈 UI에 넣지 않는다.

## PASS 조건

아래를 만족하면 PASS다.

1. 홈 첫 화면에서 가장 먼저 보이는 선택이 2개다.
   - 사진 찍었어요
   - 지금 찍으려고요

2. 사용자가 설명을 길게 읽지 않아도 어느 쪽으로 가야 할지 이해할 수 있다.

3. `사진 찍었어요` 선택 시 기존 업로드 흐름으로 자연스럽게 이어진다.

4. `지금 찍으려고요` 선택 시 기존 촬영 가이드로 자연스럽게 이어진다.

5. 기존 결과 화면 / tips / deviceType / 구조 규칙이 깨지지 않는다.

6. `npm run typecheck`
7. `npm run build`

가능하면 아래도 확인:
- `npm run test:photo-styles`
- `npx tsx tests/photo-tips-regression.test.ts`

---

## HOLD 조건

아래 중 하나라도 있으면 HOLD다.

- 홈에서 여전히 무엇을 먼저 해야 하는지 생각해야 한다.
- 2개 CTA보다 다른 기능이 더 강하게 보인다.
- 업로드나 촬영 가이드 연결이 어색하다.
- 기존 구조 규칙을 깨는 수정이 들어갔다.
- 홈 UI가 단순해졌지만 실제 흐름은 오히려 더 길어졌다.

## 결과 보고 형식

아래 형식으로만 답해라.

### 1. 변경 내용
- 어떤 파일을 수정했는지
- 어떤 방식으로 홈 2분기 구조를 만들었는지

### 2. PASS / HOLD
- PASS 또는 HOLD 한 단어로 시작
- 이유는 5줄 이내

### 3. 다음 단계 1개만 제안
다음 단계는 1개만 제안한다.

추천 우선순위는 아래 순서를 따른다.
1. 촬영 가이드에 대상 단계 추가
2. 다중 업로드 + 대표 사진 선택
3. Tone / Style / Theme 사용자 노출명 정리
4. 툴팁 노출 최소화

## Codex 실행 프롬프트

현재 상태:
- STEP A~H PASS
- STEP I는 HOLD
- 결과 화면 3단 구조, photo tips, deviceType optional, 독립 촬영 가이드는 유지 중
- 문제는 홈이 아직 초보자 기준 2분기 구조가 아니라는 점

이번 작업:
- 홈 첫 화면을 `사진 찍었어요 / 지금 찍으려고요` 2분기 구조로 정리해
- `사진 찍었어요`는 기존 업로드 흐름으로
- `지금 찍으려고요`는 기존 촬영 가이드로 연결해

수정 범위:
- 홈 진입 화면과 연결 UI
- 필요한 최소 라우팅/버튼 구조 조정

절대 건드리지 말 것:
- 결과 화면 3단 구조
- photo tips 로직
- Tone / Style / Theme 내부 구조
- deviceType contract
- 분석 로직
- 대규모 리팩토링

결과는 아래 형식으로만 줘:
1. 변경 내용
