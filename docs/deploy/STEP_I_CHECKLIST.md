# STEP I - PRE-LAUNCH CHECKLIST

## 🎯 목적
외부 배포 전 UX 흐름, 기능 안정성, 구조 유지 여부를 최종 점검한다.

---

# 1. 핵심 UX 흐름 점검

## 홈
- [ ] 첫 화면에서 선택이 2개뿐인가 (사진 찍었어요 / 지금 찍으려고요)
- [ ] 설명 없이도 행동이 이해되는가

## 촬영 가이드
- [ ] 상황 → 대상 → 기기 흐름이 자연스러운가
- [ ] 3개 팁만으로 이해 가능한가
- [ ] 설정 툴팁이 필요할 때만 보이는가

## 업로드 흐름
- [ ] 여러 장 업로드 후 대표 사진 선택이 쉬운가
- [ ] 추천 표시가 직관적인가

## 결과 화면
- [ ] 바로 쓰는 영역이 가장 먼저 보이는가
- [ ] 캡션 복사 / 이미지 저장이 바로 가능한가
- [ ] 3단 구조가 명확한가 (핵심 / 조정 / 확장)

---

# 2. Tone / Style / Theme 점검

- [ ] Tone 선택 시 결과 차이가 확실한가
- [ ] Style 적용 시 표현 방식이 명확히 변하는가
- [ ] Theme 변경 시 문장 분위기가 바뀌는가
- [ ] 세 개 개념이 사용자에게 헷갈리지 않는가

---

# 3. 사진 개선 팁 점검

- [ ] 첫 카드가 가장 중요한 행동인가
- [ ] 팁이 3~5개로 제한되어 있는가
- [ ] 문장이 짧고 행동형인가
- [ ] 타입별로 충분히 다르게 느껴지는가
- [ ] 툴팁 연결이 자연스러운가

---

# 4. deviceType UX 점검

- [ ] 선택이 필수가 아닌가
- [ ] 선택 안 해도 흐름이 자연스러운가
- [ ] 선택 시 이점이 명확한가
- [ ] 결과/가이드에 잘 반영되는가

---

# 5. 커스텀 기능 점검

- [ ] 기본 사용자에게 노출되지 않는가
- [ ] 직접 꾸미기에서만 접근 가능한가
- [ ] 기본 결과가 충분히 완성형인가

---

# 6. 성능 및 안정성

- [ ] typecheck 통과
- [ ] build 통과
- [ ] 주요 화면 에러 없음
- [ ] fallback 로직 정상 동작

---

# 7. 구조 규칙 유지 점검

- [ ] Tone / Style / Theme 분리 유지
- [ ] Frame이 Tone 내부에 포함되어 있는가
- [ ] UI에서 로직 처리하지 않는가
- [ ] content와 UI가 분리되어 있는가
- [ ] 한 파일에 기능이 몰려 있지 않은가

---

# 8. 배포 전 최종 질문

- [ ] 10초 안에 사용 가능한가
- [ ] 고민 없이 선택 가능한가
- [ ] 결과가 즉시 만족스러운가

---
