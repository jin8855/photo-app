# Codex 작업 지시서 — Photo Caption Local App 개선

## 1. 프로젝트 목적

이 프로젝트는 사용자가 사진을 업로드하면 사진 분위기를 분석하고, 인스타그램용 문구/캡션/해시태그를 생성하며, 텍스트가 올라간 소셜 이미지를 만드는 앱이다.

이번 작업의 목표는 단순한 캡션 생성기를 넘어서, **외부 베타 사용자에게 보여줄 수 있는 수준으로 UX를 정리하고**, 추가로 **사진 촬영 팁 기능**을 넣어 서비스 차별화를 만드는 것이다.

---

## 2. 이번 작업의 핵심 목표

### A. 베타 사용자용 UX 정리
현재 기능은 많지만 첫 사용자 입장에서 다음 행동이 직관적이지 않다.

따라서 아래를 반드시 반영한다.

#### 목표
1. 업로드 후 사용자가 무엇을 해야 하는지 한눈에 알 수 있어야 한다.
2. 결과 화면의 정보량을 줄이고, 행동 중심으로 재배치한다.
3. "캡션 생성" + "이미지 만들기" + "사진 개선 팁"의 3축 구조로 정리한다.

---

### B. 사진 촬영 팁 기능 추가
현재는 사진 분석 후 카피/캡션 위주다.  
여기에 **촬영 팁 / 개선 포인트**를 추가해서 서비스 가치를 높인다.

핵심 방향:
- 업로드된 사진을 보고
- "어디가 아쉬운지"
- "어떻게 다시 찍으면 좋아지는지"
- "갤럭시 S25 울트라 / S26 울트라에서 어떤 기능을 활용하면 좋은지"
를 안내한다.

---

## 3. 절대 지켜야 할 원칙

1. **기존 구조를 최대한 유지**
   - 현재 provider / repository / storage / service 경계는 유지한다.
   - 무리한 전면 재구성 금지
   - 이번 작업은 “정리 + 확장”에 집중

2. **UI 텍스트 하드코딩 최소화**
   - 가능한 한 i18n 리소스 사용
   - 새 한국어 문구는 리소스 파일에 둔다

3. **유료 OpenAI API 키 없이도 동작 가능해야 함**
   - 현재 mock/rule-based fallback 경로는 유지
   - 새 기능도 fallback 가능한 방식으로 설계

4. **모바일 우선 UX**
   - 실제 사용자는 모바일에서 캡션 복사 / 이미지 저장 / 인스타 업로드 흐름을 많이 사용할 가능성이 높다.
   - 결과 화면은 모바일 기준으로 먼저 정리한다.

---

## 4. 우선 작업 범위

### 4-1. 결과 화면 UX 재정리

#### 현재 문제
- 결과 정보가 많아 첫 사용자가 부담스러움
- 다음 행동이 눈에 안 띔
- 캡션/해시태그/리뷰/이미지 생성 기능이 섞여 있음

#### 개선 방향
업로드 결과 화면을 아래 순서로 재배치한다.

### 결과 화면 권장 구조
1. **대표 분석 요약**
   - 사진 분위기 한 줄 요약
   - 분석 스타일명
   - 짧은 리뷰

2. **즉시 행동 영역 (가장 위에 고정 또는 강조)**
   - 캡션 복사
   - 해시태그 복사
   - 이미지 만들기
   - 다시 생성하기

3. **캡션 / 문구 영역**
   - 추천 캡션 3종 정도
   - 톤 구분이 있으면 좋음
     - 기본형
     - 감성형
     - 강한형

4. **사진 개선 팁 영역**
   - 새 기능
   - 구도 / 빛 / 색감 / 기기 팁 표시

5. **상세 분석은 접을 수 있게**
   - long_review, scores, 세부 분류값 등은 기본 펼침 아님

---

### 4-2. 결과 신뢰도 보완

현재 mock visual classifier는 heuristic이라 오분류 가능성이 있다.

#### 반드시 추가할 것
- 결과가 확정값처럼 보이지 않도록 보완
- 아래 중 하나 이상 반영
  1. 분석 확신도(Confidence) 표시
  2. “다른 스타일로 다시 보기” 버튼
  3. “결과가 어색하면 다시 생성해보세요” 안내

#### 주의
- 사용자 신뢰를 잃지 않도록 너무 과장된 AI 표현 금지

---

### 4-3. 촬영 팁 기능 추가

## 새 기능 이름(가칭)
- Photo Improvement Tips
- Shooting Tips
- 사진 개선 팁
- 촬영 가이드

한국어 서비스상으로는 **“사진 개선 팁”**이 가장 무난하다.

---

## 5. 촬영 팁 기능 상세 설계

### 기능 목표
업로드된 사진에 대해 다음을 제공한다.

1. 구도 팁
2. 빛/노출 팁
3. 색감/분위기 팁
4. 기기별 팁
5. 다시 찍을 때의 실전 조언

---

### 출력 예시 형식
```json
{
  "composition_tip": "피사체가 중앙에 몰려 있어 답답해 보일 수 있습니다. 살짝 옆으로 두고 여백을 주면 더 자연스럽습니다.",
  "lighting_tip": "밝은 영역이 강해서 하이라이트가 조금 날아간 느낌입니다. 노출을 약간 낮춰 찍어보세요.",
  "color_tip": "전체 톤이 탁한 편이라 밝은 시간대나 따뜻한 색감을 살리면 분위기가 더 좋아집니다.",
  "device_tip": "갤럭시 울트라에서는 3배/5배 줌을 활용해 배경 압축감을 주면 피사체가 더 살아납니다.",
  "retake_tip": "한 걸음 뒤로 물러난 뒤 3분할 구도로 다시 찍어보면 훨씬 안정적입니다."
}
```

---

### 분석 입력값 제안
가능한 범위 내에서 아래 데이터를 활용한다.

- `scene_type`
- `mood_category`
- `photo_style_type`
- 밝기 추정
- 대비 추정
- 채도 추정
- 중앙 집중도 / 좌우 여백 정도 등 간단 휴리스틱
- blur / sharpness 추정 가능하면 활용

---

### 구현 방식
새 service 레이어를 추가한다.

권장 구조:

```txt
services/tips/
  photo-tip-service.ts
  providers/
    mock-photo-tip-provider.ts
  strategies/
    action-speed-tip-strategy.ts
    travel-tip-strategy.ts
    portrait-tip-strategy.ts
    landscape-tip-strategy.ts
    lowlight-tip-strategy.ts
```

#### 최소 구현 원칙
- 처음부터 복잡한 CV 모델 만들지 말 것
- 현재 프로젝트 방향에 맞게 rule-based + simple image heuristics 기반으로 시작
- 추후 AI vision provider가 붙더라도 교체 가능하게 contract 설계

## 6. 갤럭시 S25 Ultra / S26 Ultra 팁 반영 방향

이번 1차에서는 “실제 기기 자동 판별”보다  
**사용자가 설정 또는 선택한 기기 기준으로 팁 노출**하는 방식이 현실적이다.

### 추천 방식
- 설정값 또는 드롭다운 추가
  - 기본값: Galaxy Ultra
  - 세부: S25 Ultra / S26 Ultra / 일반 스마트폰

### 기기별 팁 예시
#### Galaxy S25 Ultra / S26 Ultra 공통
- 3배/5배 줌 활용
- 야간 촬영 시 손떨림 줄이기
- 고배율 줌에서는 손 고정 중요
- 역광 시 HDR 활용
- 음식/소품은 너무 가까운 초광각보다 2배 이상이 자연스러움

#### 저조도
- Night 모드 적극 활용
- 흔들림이 있으면 벽/테이블 지지
- 움직이는 피사체는 자동 야간보다 밝은 장소 이동이 더 효과적

#### 인물/소품
- 배경 흐림 과도할 경우 거리 조절
- 얼굴/주제 피사체는 중앙 고정보다 살짝 오프셋
- 강한 실내 조명에서는 노출을 조금만 낮추기

### 주의
- 실제 기기 스펙을 단정적으로 과장하지 말 것
- 설정명/UI명은 앱 내 표현을 너무 세세하게 박지 말고 일반화된 조언 위주로

## 8. 저장/히스토리 정책

촬영 팁은 분석 결과에 종속되므로 다음 원칙을 권장한다.

### 권장
- 사용자가 명시적으로 결과를 저장할 때만 persistence
- 실시간 미리보기 단계에서는 자동 저장 최소화

### 이유
- 베타 단계에서는 불필요한 저장 증가를 줄이는 것이 더 중요
- history가 너무 많은 중간 결과로 오염되면 UX가 나빠짐

---

## 9. 가장 먼저 손볼 우선순위

### P1
1. 결과 화면의 다음 행동 CTA 정리
2. 캡션/해시태그/이미지 만들기 버튼 가시성 강화
3. 사진 개선 팁 섹션 추가
4. mock 분석 결과의 확신도/재생성 UX 보완

### P2
5. 분석 상세 정보 접기/펼치기
6. 모바일 레이아웃 정리
7. history에서 “다시 편집 / 다시 생성” 흐름 강화

### P3
8. 기기 선택값 저장
9. tip strategy 세분화
10. 추후 real AI provider 붙일 수 있도록 contract 정리

---

## 10. 구현 시 주의사항

1. 기존 action/speed 카피 톤은 유지하되, 촬영 팁은 별도 톤으로 분리
2. 카피 엔진과 팁 엔진의 책임을 섞지 말 것
3. provider factory를 오염시키지 말고 새 provider 등록은 규칙적으로 할 것
4. DB provider / storage provider의 기존 분리 철학 유지
5. beta-ready 관점에서 “정확도 100%”보다 “UX 신뢰도 80% 이상”을 목표로 할 것
