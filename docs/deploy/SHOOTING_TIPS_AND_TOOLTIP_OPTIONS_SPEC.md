# SHOOTING_TIPS_AND_TOOLTIP_OPTIONS_SPEC.md

## 목적
촬영 팁 영역에서
- 기본 화면에는 짧고 행동형 팁만 보이고
- 자세히 보기 또는 툴팁에서는 더 구체적인 설명값이 보이도록

옵션 구조와 노출 규칙을 먼저 고정한다.

이번 문서는 Codex가 촬영 팁 UI를 다시 맞출 때 사용하는 **정의서**다.

---

# 1. 기본 원칙

촬영 팁은 2단 구조로 간다.

## A. 기본 노출
사용자가 바로 행동할 수 있는 짧은 팁만 보여준다.

예:
- 한 걸음 더 가까이 찍기
- 밝기 먼저 맞추기
- 색은 한 톤만 살리기

## B. 확장 노출
자세히 보기 / 툴팁 / 펼치기에서
왜 그런지, 어떻게 하면 되는지, 언제 쓰면 좋은지를 보여준다.

즉:
- 기본 = 행동형 한 줄
- 확장 = 설명형 2~4줄

---

# 2. 촬영 팁 카드 구조

각 촬영 팁은 아래 구조를 가진다.

## 필수
1. tipTitle
2. tipShort
3. tipCategory

## 확장
4. tipDetail
5. tipWhy
6. tipWhen
7. tipExample
8. relatedSettingHint (선택)
9. deviceHint (선택)

---

# 3. 필드 정의

## 3-1. tipTitle
### 의미
카드 상단의 짧은 제목

### 예시
- 한 걸음 더 찍기
- 밝기 먼저 맞추기
- 배경 정리하기

### 노출 위치
- 카드 상단
- 항상 보임

---

## 3-2. tipShort
### 의미
기본 화면에서 바로 보이는 행동형 한 줄 설명

### 예시
- 피사체를 조금 더 크게 담아보세요.
- 밝은 곳에 초점을 먼저 맞추세요.
- 주변 물건을 한두 개만 정리해보세요.

### 노출 위치
- 카드 본문
- 항상 보임

### 규칙
- 짧게
- 바로 행동 가능해야 함
- 추상적 설명 금지

---

## 3-3. tipCategory
### 의미
팁의 성격을 구분하는 짧은 카테고리 라벨

### 예시
- 구도
- 밝기
- 색감
- 거리
- 기기 힌트
- 배경
- 인물
- 음식
- 제품

### 노출 위치
- 카드 상단 배지 또는 칩
- 항상 보임 가능

---

## 3-4. tipDetail
### 의미
자세히 보기에서 보여주는 핵심 설명

### 예시
- 화면의 절반 이상을 주제에 쓰면 전달력이 더 강해집니다.
- 역광일 때는 밝은 하늘보다 피사체 얼굴 밝기를 먼저 확인하세요.

### 노출 위치
- 자세히 보기 / 펼침 / 툴팁

### 규칙
- 1~3문장
- 행동 이유와 방법을 함께 설명

---

## 3-5. tipWhy
### 의미
왜 이 팁이 중요한지 설명

### 예시
- 배경이 복잡하면 사진의 중심이 약해집니다.
- 밝기 균형이 맞아야 후반 보정이 쉬워집니다.

### 노출 위치
- 자세히 보기 / 툴팁 안

### 규칙
- 사용자가 납득할 수 있게 짧게

---

## 3-6. tipWhen
### 의미
언제 이 팁이 특히 유효한지 설명

### 예시
- 실내 음식 사진일 때
- 역광 인물 사진일 때
- 제품을 책상 위에서 찍을 때

### 노출 위치
- 자세히 보기 / 툴팁 안

---

## 3-7. tipExample
### 의미
간단한 예시 또는 적용 방식

### 예시
- 꽃 사진이라면 배경 하늘 비중을 줄이고 꽃 크기를 더 키워보세요.
- 컵 사진이라면 컵 손잡이가 반쯤 보이게 두면 형태가 잘 살아납니다.

### 노출 위치
- 자세히 보기 / 툴팁 안

---

## 3-8. relatedSettingHint
### 의미
촬영 설정과 연결되는 힌트

### 예시
- 노출을 -0.3 정도 낮추면 색이 더 진하게 보일 수 있습니다.
- 인물 모드가 있으면 배경 정리에 도움이 됩니다.

### 노출 위치
- 설정 툴팁
- deviceType 힌트 영역
- 자세히 보기

### 주의
- 모든 팁에 필수는 아님
- 설정값은 너무 기술적으로 길지 않게

---

## 3-9. deviceHint
### 의미
기기별 특이 팁

### 예시
- 갤럭시에서는 화면을 길게 눌러 초점과 밝기를 같이 잡아보세요.
- 아이폰에서는 라이브 포토를 끄면 구도 고정이 쉬워집니다.

### 노출 위치
- 자세히 보기 / deviceType 연결 영역

### 주의
- deviceType 선택 시에만 강조
- 기본 화면에는 과노출 금지

---

# 4. 기본 화면 노출 규칙

기본 화면에서는 아래만 보여준다.

- tipCategory
- tipTitle
- tipShort

즉, 사용자는 카드만 보고 바로 행동할 수 있어야 한다.

## 기본 화면에 하지 말 것
- 긴 설명 전체 노출
- 설정값 숫자 과다 노출
- 예시문 여러 개 나열
- 기기별 설명 과다 노출

---

# 5. 자세히 보기 / 툴팁 노출 규칙

자세히 보기 또는 툴팁에서는 아래를 보여준다.

- tipDetail
- tipWhy
- tipWhen
- tipExample
- relatedSettingHint (있을 때만)
- deviceHint (있을 때만)

---

# 6. UI 방식 권장

## 방법 A. 카드 펼치기
- 기본 카드 아래 `자세히 보기`
- 클릭 시 카드 내부 확장

## 방법 B. 툴팁 / 정보 아이콘
- 짧은 정보만 볼 때 적합
- 데스크톱에서 좋음

## 방법 C. 바텀시트 / 모달
- 모바일에서 좋음
- 여러 상세 값이 있을 때 적합

### 권장
- 기본: `자세히 보기`
- 보조: 설정 아이콘 툴팁
- 모바일: 바텀시트 가능

---

# 7. 촬영 팁 우선순위 규칙

## 첫 카드
가장 중요한 행동 1개

## 나머지 카드
3~5개 안에서 보조 팁

### 규칙
- 첫 카드는 무조건 즉시 행동 가능한 문장
- 상세 설명은 확장 노출에만 둔다

---

# 8. 예시 구조

## 카드 예시 1
- tipCategory: 구도
- tipTitle: 한 걸음 더 가까이
- tipShort: 피사체를 조금 더 크게 담아보세요.
- tipDetail: 화면 안에서 주제가 차지하는 비중이 커지면 사진의 전달력이 더 좋아집니다.
- tipWhy: 배경이 너무 많으면 시선이 분산됩니다.
- tipWhen: 꽃, 음식, 소품 사진일 때
- tipExample: 꽃잎 가장자리까지 화면 안에 꽉 차게 담아보세요.
- relatedSettingHint: 없음
- deviceHint: 없음

## 카드 예시 2
- tipCategory: 밝기
- tipTitle: 밝기 먼저 맞추기
- tipShort: 너무 밝은 부분보다 주제가 잘 보이게 맞춰보세요.
- tipDetail: 하늘이나 조명이 너무 밝으면 주제가 어둡게 묻힐 수 있습니다.
- tipWhy: 주제가 보여야 문구와 이미지도 잘 어울립니다.
- tipWhen: 역광, 창가, 실내 조명 아래
- tipExample: 얼굴이나 음식 위쪽 밝기를 먼저 확인하세요.
- relatedSettingHint: 노출을 약간 낮추면 색이 더 진하게 보일 수 있습니다.
- deviceHint: 갤럭시에서는 길게 눌러 초점과 밝기를 같이 잡아보세요.

# 9. Codex 적용 기준

이번 작업에서 Codex는 촬영 팁을 아래처럼 정리해야 한다.

## 기본 카드
- category + title + short

## 확장 정보
- detail + why + when + example + optional hints

## 반영 목표
1. 기본 화면은 짧고 간결하게
2. 자세히 보기나 툴팁에서 더 구체적인 값 제공
3. deviceType이 있을 때만 기기 힌트 강화
4. 설정 힌트는 과하게 기술적으로 보이지 않게

---

# 10. 금지 규칙

- 모든 상세 설명을 기본 화면에 다 펼치지 말 것
- 기본 팁 카드에 숫자/설정값을 과하게 넣지 말 것
- 팁 제목과 설명이 같은 말 반복이 되지 않게 할 것
- 사진 변경 / 문구 변경 영역과 촬영 팁 영역을 섞지 말 것

---

# 11. Codex 실행 프롬프트

현재 문제:
- 촬영 팁의 상세 값(자세히 보기 / 툴팁용 설명)이 화면에 명확히 반영되지 않는다
- 기본 팁과 상세 팁의 노출 기준이 고정되어 있지 않다

이번 작업:
- 첨부된 촬영 팁 옵션 정의서를 기준으로 팁 카드 구조를 다시 맞춰
- 기본 화면에는 짧은 행동형 팁만 노출하고
- 자세히 보기 / 툴팁에서는 상세 설명값이 보이게 정리해

핵심 적용:
1. 기본 카드
   - category
   - title
   - short

2. 확장 정보
   - detail
   - why
   - when
   - example
   - relatedSettingHint
   - deviceHint

3. UI
   - 기본은 짧게
   - 자세히 보기 또는 툴팁에서 상세값 표시

절대 건드리지 말 것:
- 분석 로직
- 생성 로직
- step 구조

결과:
1. 어떤 팁 구조로 반영했는지
2. PASS / HOLD
3. 다음 단계 1개
