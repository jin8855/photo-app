# TASK_STEP_J2_SHOOTING_GUIDE_ADD_SUBJECT_STEP.md

## 목적
STEP J 1차에서 홈 2분기 구조를 PASS로 정리했으므로,
다음으로 촬영 가이드의 핵심 HOLD를 해소한다.

이번 작업의 목표는 촬영 가이드를
`상황 -> 기기` 구조에서
`상황 -> 대상 -> 기기` 구조로 확장해
초보 사용자가 더 정확한 가이드를 받도록 만드는 것이다.

---

## 현재 상태

현재까지 PASS 상태:
- 홈 첫 진입 2분기 구조
- 결과 화면 3단 구조
- photo tips 3~5개 제한 및 우선순위 정렬
- deviceType optional 흐름
- 독립 촬영 가이드 진입

현재 남은 UX HOLD 중 하나:
- 촬영 가이드에 `대상` 단계가 없음
- 체크리스트 기준은 `상황 -> 대상 -> 기기`
- 현재는 `상황 -> 기기`라서 가이드 정밀도가 부족함

## 이번 단계 범위

### 반드시 할 것
1. 촬영 가이드에 `대상` 선택 단계 추가
2. 흐름을 아래 순서로 맞춘다
   - 상황
   - 대상
   - 기기
3. 대상 선택값이 가이드 결과에 반영되도록 연결
4. 기존 독립 촬영 가이드 진입 구조는 유지

### 이번 단계에서 허용되는 최소 확장
- 대상 옵션 4개 내외로 시작
- 텍스트/카피 미세 조정
- 현재 가이드 문구에서 대상별 차이를 분기

## 구조 규칙 (절대 유지)

다음 구조 규칙은 유지한다.

- Tone / Style / Theme 3축 유지
- Frame은 Tone 내부
- features / domains / content 분리
- UI는 로직을 가지지 않음
- content 직접 import 금지
- 촬영 가이드 UI가 분석 로직을 직접 품지 않음

## 대상 단계 설계 기준

### 대상은 너무 많게 시작하지 말 것
1차는 아래 정도로 작게 시작하는 것을 권장한다.

- 인물
- 음식
- 풍경
- 제품

필요하면 현재 프로젝트의 기존 category 체계와 맞는 이름으로 조정 가능하다.
단, 개수는 처음부터 많게 늘리지 않는다.

### 선택 기준
사용자는 “무엇을 찍는지”를 빠르게 고를 수 있어야 한다.
전문 용어보다 일상 용어를 우선한다.

---

## UX 요구사항

### A. 흐름이 길어 보이지 않아야 함
단계는 늘어나지만 부담은 늘어나지 않아야 한다.

즉:
- 질문은 명확하게
- 선택지는 짧게
- 한 번에 하나만 고르게

### B. 대상 단계는 상황 다음에 와야 함
현재 체크리스트 기준을 맞춘다.

순서:
1. 상황
2. 대상
3. 기기

### C. 기존 기기 단계는 유지
deviceType은 여전히 optional이어도 된다.
단, 대상 선택 후 기기 선택으로 자연스럽게 이어져야 한다.

### D. 결과 가이드 차이가 느껴져야 함
대상을 추가했는데 결과 문구가 거의 같으면 HOLD다.
최소한 사용자가 “아, 대상 따라 팁이 달라졌네” 정도는 느껴야 한다.

## 구현 가이드

### 1. 최소 변경 우선
촬영 가이드 전체를 새로 만들지 말고,
현재 흐름에 `대상` 단계를 끼워 넣는 방향으로 간다.

### 2. 기존 구조 재사용
가능하면 기존:
- 라우트
- 가이드 화면
- 상태 관리
- 기기 선택 구조
를 재사용한다.

### 3. 콘텐츠 분리 유지
대상별 가이드 차이는 content/domain 계층에서 관리하고,
UI는 선택과 표시만 담당한다.

### 4. 과한 일반화 금지
처음부터 모든 대상/상황 조합을 일반화하려 하지 말고,
작은 명시적 분기부터 시작한다.

## PASS 조건

아래를 만족하면 PASS다.

1. 촬영 가이드 흐름이 `상황 -> 대상 -> 기기`가 된다.
2. 대상 단계가 독립적으로 보이고, 사용자가 이해하기 쉽다.
3. 대상 선택에 따라 가이드 문구 차이가 실제로 느껴진다.
4. 기존 `/shooting-guide` 진입이 유지된다.
5. deviceType optional 흐름이 깨지지 않는다.
6. 결과 화면 / photo tips / 홈 2분기 구조가 깨지지 않는다.

검증:
- `npm run typecheck`
- `npm run build`

가능하면 추가 확인:
- `npm run test:photo-styles`
- `npx tsx tests/photo-tips-regression.test.ts`

---

## HOLD 조건

아래 중 하나라도 있으면 HOLD다.

- 대상 단계가 추가됐지만 흐름이 오히려 복잡해졌다.
- 대상 선택이 있어도 결과 가이드 차이가 거의 없다.
- UI가 로직을 직접 품게 되었다.
- 기존 홈 2분기 구조나 업로드 흐름에 영향이 갔다.
- 기기 단계 연결이 어색하거나 끊긴다.
